# Respuestas a la Practica 1 Ingenieria de Software 2025

1) Un requerimiento (o requisito) es una descripción clara de lo que el sistema debe hacer o como debe comportarse, con el fin de cumplir con las necesidades o expectativas del cliente o usuario final. Puede referirse tanto a funciones especificas como a condiciones que el sistema debe cumplir y/o respetar (restricciones).

2) Requerimientos funcionales: lo que el sistema DEBE hacer, for example, permitir al usuario iniciar sesión.

Requerimientos no funcionales: COMO DEBE hacerlo, for example, la pagina debe cargar en menos de 2 segundos.

3) Los stakeholders (o partes interesadas) son todas aquellas personas, grupos u organizaciones que se ven afectadas por el sistema de manera directa o indirectamente, ya sea de forma positiva o negativa. Los mas comunes son, usuarios finales, ingenieros y/o desarrolladores, gerentes o directivos y personas como soporte, patrocinadores, entre otros.

4) Las principales fuentes de información(requerimientos) son:
* DOCUMENTACION: manuales, informes, procesos actuales o cualquier documento que describa el sistema actual.
* STAKEHOLDERS: usuarios, clientes, gerentes, desarrolladores y cualquier persona involucrada que pueda aportar requerimientos, expectativas y restricciones.
* ESPECIFICACION DE SISTEMAS SIMILARES: analisis de soluciones previas o de competencia que pueda servir como referencia.

5) Los puntos de vista que se puede encontrar en el desarrollo de un sistema de software son: INTERACTUADORES (personas o organizaciones que utilizan el sistema de manera directa, influye en los requerimientos), INDIRECTO (personas o organizaciones que no utilizan el sistema, pero influye en los requerimientos) y DOMINIO (características y restricciones del dominio que influye en los requerimientos)
